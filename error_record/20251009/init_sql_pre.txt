 紧急情况下，内部系统没有dos2unix命令，怎么用linux命令给脚本执行替换分隔符？例如当前的init_sql.sh脚本

● 几种常用方法：

  1. sed（最常用）
  sed -i 's/\r$//' /path/to/init_sql.sh

  2. tr
  tr -d '\r' < init_sql.sh > init_sql_tmp.sh && mv init_sql_tmp.sh init_sql.sh

  3. perl
  perl -pi -e 's/\r$//' init_sql.sh

  4. awk
  awk '{ sub(/\r$/,""); print }' init_sql.sh > tmp.sh && mv tmp.sh init_sql.sh

  5. vi/vim（非交互式）
  vi -c 'set ff=unix' -c 'wq' init_sql.sh

  ---
  推荐用 sed -i 's/\r$//'，几乎所有 Linux 系统都有，简单直接。

  如果 sed -i 报错（某些旧系统），可以用：
  sed 's/\r$//' init_sql.sh > tmp.sh && mv tmp.sh init_sql.sh
