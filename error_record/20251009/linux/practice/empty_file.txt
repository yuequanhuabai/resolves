  练习 1：观察区别

  创建一个文件并写入内容，然后分别用 > 和 touch 操作它，观察区别：

  echo "hello world" > test.txt
  cat test.txt        # 看内容
  touch test.txt
  cat test.txt        # 内容还在吗？
  > test.txt
  cat test.txt        # 内容还在吗？

  问题：解释你观察到的现象。

  练习 5：安全写入模式

  写一个脚本，模拟"安全写入"流程：

  1. 用 : > 初始化一个临时文件 output.tmp
  2. 往临时文件中写入 5 行数据（用 >> 追加）
  3. 写入完成后，用 mv 原子替换为最终文件 output.txt
  4. 加入 trap 机制：如果脚本中途被中断（Ctrl+C），自动清理 output.tmp

  问题：为什么不直接用 > output.txt 写入，而要先写临时文件再 mv？
