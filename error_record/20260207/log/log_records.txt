  前置概念：两类日志文件

  - archive 文件：$LOG_DIR/archive/ 下，文件名含日期（如 xxx.2026-04-09.xxx.log）
  - server 日志：$LOG_DIR/server_start.log 和 server_stop.log，无日期，只有一份

  ---
  临界条件：昨天的归档

  server 日志只会打进"昨天"的包，这是关键。分四种场景：

  ┌──────┬─────────────────────────┬───────────────────┬────────────────────────────────┐
  │ 场景 │ archive/ 有昨天的文件？ │ server 日志非空？ │            打包内容            │
  ├──────┼─────────────────────────┼───────────────────┼────────────────────────────────┤
  │ 1    │ 是                      │ 是                │ archive 昨天文件 + server 日志 │
  ├──────┼─────────────────────────┼───────────────────┼────────────────────────────────┤
  │ 2    │ 是                      │ 否/不存在         │ 仅 archive 昨天文件            │
  ├──────┼─────────────────────────┼───────────────────┼────────────────────────────────┤
  │ 3    │ 否                      │ 是                │ 仅 server 日志                 │
  ├──────┼─────────────────────────┼───────────────────┼────────────────────────────────┤
  │ 4    │ 否                      │ 否/不存在         │ 不打包，跳过                   │
  └──────┴─────────────────────────┴───────────────────┴────────────────────────────────┘

  ▎ 场景3是关键边界：archive/ 里没有昨天的文件，但 server 日志有内容，脚本仍会为昨天生成一个只含 server 日志的包。

  ---
  正常逻辑：昨天之前的历史日期

  历史日期只来自 archive/ 文件名的扫描，server 日志永远不会打进历史日期的包。只有两种场景：

  ┌──────┬──────────────────────────────────────────────┬─────────────────────────┐
  │ 场景 │                     说明                     │        打包内容         │
  ├──────┼──────────────────────────────────────────────┼─────────────────────────┤
  │ 1    │ 该日期在 archive/ 下有文件，且 tar.gz 不存在 │ 仅该日期的 archive 文件 │
  ├──────┼──────────────────────────────────────────────┼─────────────────────────┤
  │ 2    │ 该日期的 tar.gz 已存在                       │ 跳过（幂等）            │
  └──────┴──────────────────────────────────────────────┴─────────────────────────┘

 核心归档逻辑：

  ---
  第一步：收集待归档日期

  扫描 archive/ 下所有非空 .log 文件，从文件名中提取日期，按日期去重分组，筛选出 <= 昨天 且在 ARCHIVE_DIR 下没有对应 tar.gz 的日期，组成待归档日期列表。

  ---
  第二步：处理 server 日志

  检查 server_start.log、server_stop.log 是否存在且非空：

  - 如果昨天已在待归档日期列表中：直接将非空的 server 日志加入昨天的归档文件列表
  - 如果昨天不在待归档日期列表中，且昨天的 tar.gz 也不存在：
    - server 日志有任意一个非空 → 手动将昨天加入日期列表，并将非空的 server 日志加入昨天的归档文件列表
    - server 日志全部为空或不存在 → 不做任何处理

  ---
  第三步：按日期逐个打包

  对日期列表中的每个日期：

  1. 收集 archive/ 下该日期对应的非空 .log 文件
  2. 如果是昨天，额外加入非空的 server 日志
  3. 执行 tar -czf 打包到 ARCHIVE_DIR
  4. 删除 archive/ 下已打包的源 .log 文件（server 日志不在此步删除）

  ---
  第四步：清空 server 日志

  昨天的包打完后，将 server_start.log、server_stop.log 清空（truncate，保留文件本身）。
