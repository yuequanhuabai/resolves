pap-log-archive.sh 脚本逻辑说明
=====================================

日志源：/opt/project/PAP/IHUB/back-end/logs/IHUB_*.log（活跃日志）
         /opt/project/PAP/IHUB/back-end/logs/history/IHUB_*.{yyyy-MM-dd}.*.log（logback 当天滚动日志）
归档目标：/LOG/{主机名}.IHUB-PAP.applog.{YYYYMMDD}.tar.gz
错误日志：/LOG/archive_error.log
运行日志：/LOG/archive.log

一、安全保障
-----------
1. set -e           任意命令失败立即退出，防止后续步骤在异常状态下继续执行
2. flock 文件锁     通过 /tmp/archive_IHUB-PAP.lock 防止多个实例同时运行
3. 权限检查         检查日志目录可读、归档目录存在且可写
4. 幂等检查         当天归档文件已存在则跳过，避免重复归档覆盖
5. 磁盘空间检查     分别检查日志目录（cp 副本空间）和归档目录（tar.gz 空间）

二、归档流程
-----------
步骤1：查找日志文件
  - find 查找 logs/ 下所有非空的 IHUB_*.log 活跃日志文件
  - find 查找 logs/history/ 下当天的 IHUB_*.{yyyy-MM-dd}.*.log 滚动日志文件
  - 使用 -print0 + 数组存储，安全处理含空格或特殊字符的文件名
  - 两类文件都为空则记录日志并退出

步骤2：磁盘空间检查
  - du -sk 计算活跃日志 + history 当天日志的总大小（KB）
  - df -k 获取日志目录可用空间，不足则报错退出
  - df -k 获取归档目录可用空间，不足则报错退出

步骤3：copytruncate 轮转（仅活跃日志）
  - 第一轮循环：将每个 IHUB_*.log 复制为 IHUB_*.log-{YYYYMMDD}（全部复制成功后才进入下一步）
  - 第二轮循环：清空所有原活跃日志文件
  - 分两轮执行保证原子性：任一 cp 失败时原文件不会被清空
  - history 文件不做复制和清空，由 logback 自行管理

步骤4：打包归档
  - find 查找当天产生的 IHUB_*.log-{YYYYMMDD} 轮转文件
  - 将轮转文件 + history 当天文件一起打包为 /LOG/{主机名}.IHUB-PAP.applog.{YYYYMMDD}.tar.gz
  - tar 包内结构：轮转文件在根目录，history 文件保留 history/ 子目录
  - tar 成功后只删除轮转临时文件；history 文件不删除，由 logback 的 maxHistory 和 totalSizeCap 自行清理
  - tar 失败则保留所有临时文件可手动恢复

步骤5：记录归档日志
  - 向 /LOG/archive.log 写入一条归档完成记录

三、错误处理
-----------
  - 所有错误通过 log_error 函数统一处理
  - 错误信息格式：{时间} - [IHUB-PAP] ERROR: {错误描述}
  - 写入 /LOG/archive_error.log 后退出脚本

四、与 logback 的协作关系
-------------------------
  - logback 写入 5 个分级活跃日志：IHUB_ALL.log、IHUB_INFO.log、IHUB_DEBUG.log、IHUB_WARN.log、IHUB_ERROR.log
  - 当单个文件超过 maxFileSize 或跨天时，logback 自动滚动到 history/ 目录
  - 本脚本归档活跃日志 + history 当天日志，确保不遗漏因 logback 滚动而转移到 history/ 的日志
  - history/ 目录的清理完全由 logback 负责（maxHistory / totalSizeCap），脚本不干预
