 JWT 分支 vs Session 分支 整体对比

  核心理念差异

  ┌──────────────────┬───────────────────────┬──────────────────────────────────────┐
  │                  │     Session 分支      │               JWT 分支               │
  ├──────────────────┼───────────────────────┼──────────────────────────────────────┤
  │ 认证状态存在哪   │ 服务端内存（Session） │ 客户端（Token 自身携带）             │
  ├──────────────────┼───────────────────────┼──────────────────────────────────────┤
  │ 服务端是否有状态 │ 有状态                │ 无状态                               │
  ├──────────────────┼───────────────────────┼──────────────────────────────────────┤
  │ 认证信息载体     │ Cookie（JSESSIONID）  │ Authorization Header（Bearer Token） │
  └──────────────────┴───────────────────────┴──────────────────────────────────────┘

  ---
  后端差异

  ┌──────────────┬─────────────────────────────────────┬──────────────────────────┐
  │              │            Session 分支             │         JWT 分支         │
  ├──────────────┼─────────────────────────────────────┼──────────────────────────┤
  │ Session 策略 │ IF_REQUIRED                         │ STATELESS                │
  ├──────────────┼─────────────────────────────────────┼──────────────────────────┤
  │ 登录响应     │ 只返回 username                     │ 返回 token + username    │
  ├──────────────┼─────────────────────────────────────┼──────────────────────────┤
  │ 登出处理     │ Spring 自动销毁 Session + 清 Cookie │ 后端什么都不做           │
  ├──────────────┼─────────────────────────────────────┼──────────────────────────┤
  │ RememberMe   │ 手动延长 Session TTL（7天）         │ Token 有效期写在配置文件 │
  ├──────────────┼─────────────────────────────────────┼──────────────────────────┤
  │ CORS         │ 必须 allowCredentials(true)         │ 不需要                   │
  ├──────────────┼─────────────────────────────────────┼──────────────────────────┤
  │ 独有类       │ 无                                  │ JwtUtil + JwtAuthFilter  │
  └──────────────┴─────────────────────────────────────┴──────────────────────────┘

  ---
  前端差异

  ┌──────────────────────┬─────────────────────────────────┬──────────────────────────────────┐
  │                      │          Session 分支           │             JWT 分支             │
  ├──────────────────────┼─────────────────────────────────┼──────────────────────────────────┤
  │ Axios 配置           │ withCredentials: true           │ 请求拦截器注入 Bearer Token      │
  ├──────────────────────┼─────────────────────────────────┼──────────────────────────────────┤
  │ 401 处理             │ 无需处理                        │ 响应拦截器统一处理               │
  ├──────────────────────┼─────────────────────────────────┼──────────────────────────────────┤
  │ 本地存储             │ 不用 localStorage               │ token + username 存 localStorage │
  ├──────────────────────┼─────────────────────────────────┼──────────────────────────────────┤
  │ 认证恢复（刷新页面） │ 异步请求后端 fetchCurrentUser() │ 同步读 localStorage              │
  ├──────────────────────┼─────────────────────────────────┼──────────────────────────────────┤
  │ 路由守卫             │ async，需等待网络请求           │ sync，无需网络请求               │
  ├──────────────────────┼─────────────────────────────────┼──────────────────────────────────┤
  │ 登录表单             │ 有"记住我"复选框                │ 没有                             │
  ├──────────────────────┼─────────────────────────────────┼──────────────────────────────────┤
  │ logout               │ 异步，调 API 通知后端           │ 同步，只清 localStorage          │
  └──────────────────────┴─────────────────────────────────┴──────────────────────────────────┘

  ---
  各自的代价

  Session 分支多做的事：
  - 后端维护 Session 存储
  - 前端每次刷新页面要请求后端确认身份（多一次网络请求）
  - CORS 必须允许 Cookie 凭证

  JWT 分支多做的事：
  - 后端多写 JwtUtil + JwtAuthFilter
  - 前端多写请求/响应拦截器、localStorage 存取、Token 注入

  ---
  一句话总结

  Session 把复杂度压在服务端（存状态、管生命周期），前端省事但刷新页面要回调后端确认；JWT 把复杂度压在客户端（存
  Token、自己注入、自己恢复），后端完全无状态，但前端要多管的东西更多。
