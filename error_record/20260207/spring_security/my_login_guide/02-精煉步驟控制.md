# 02 — 精煉步驟控制（路線圖）

> 本文檔回答：**「我從零開始，要做哪幾件大事？順序是什麼？」**
> 不講代碼，只給 checklist。每一步背後的細節在 `03-執行步驟.md`。

---

## 整體節奏

把整個搭建拆成 **6 個階段，21 個步驟**。
建議**一個階段做完就完整跑一遍**，確認沒問題再進下一階段。

```
階段 1：環境準備           ─ 沒環境啥都做不了
階段 2：項目骨架           ─ 能啟動一個空殼 Spring Boot
階段 3：數據層             ─ 數據庫連得上，能讀寫表
階段 4：安全與登錄         ─ 登錄頁能用，密碼能驗
階段 5：頁面與接口         ─ Dashboard、用戶列表、API
階段 6：完善與驗證         ─ Remember-me、整體聯調、踩坑修復
```

---

## 階段 1：環境準備（Done = 命令行能 `mvn -v`、`java -version`、能連上數據庫）

- [ ] **1.1 確認 JDK 17+**（Spring Boot 3.x 強制要求）
- [ ] **1.2 確認 Maven 已安裝**（或用 IDE 自帶）
- [ ] **1.3 準備好數據庫**（SQL Server / MySQL 二選一，本指南以 SQL Server 為例）
- [ ] **1.4 用數據庫客戶端（DBeaver、Navicat、SSMS 等）能連上你的數據庫**

> 卡住了看 04-踩坑記錄.md 中的 **「環境類問題」** 段。

---

## 階段 2：項目骨架（Done = `mvn spring-boot:run` 能啟動，控制檯沒紅錯）

- [ ] **2.1 用 [start.spring.io](https://start.spring.io) 生成項目骨架**
  - 選 Maven、Java 17+、Spring Boot 3.2.x
  - 依賴選：Spring Web、Spring Security、Thymeleaf、Spring Data JPA、Lombok、（你的數據庫驅動）
  - groupId / artifactId 自定（建議 `com.example` / `test-login-back`）
- [ ] **2.2 解壓導入 IDE**（IntelliJ IDEA / VS Code 均可）
- [ ] **2.3 寫一個最簡 `Application` 啟動類，跑起來**
- [ ] **2.4 配置 `application.yml`** 的 `server.port`（暫時不配數據源，下一階段配）

⚠️ 注意：此時直接啟動會因為「沒有數據源」報錯。  
有兩個選擇：① 先進階段 3 配好數據庫；② 在啟動類上臨時加 `@SpringBootApplication(exclude=DataSourceAutoConfiguration.class)`。**推薦選 ①**。

---

## 階段 3：數據層（Done = 啟動後 JPA 不報錯，sys_user 表存在且有測試數據）

- [ ] **3.1 數據庫建庫建表**
  - 庫名 `test_login`
  - 表 `sys_user`：`id`(主鍵自增) / `username`(唯一) / `password` / `enabled`
- [ ] **3.2 在 `application.yml` 配置數據源**（URL、用戶名、密碼、driver-class-name）
- [ ] **3.3 配置 JPA**
  - `ddl-auto: none`（自己控制建表，別讓 Hibernate 自動改表結構）
  - `show-sql: true`（打印 SQL，方便調試）
  - `database-platform`（指定方言，如 SQLServerDialect）
- [ ] **3.4 寫 Entity `SysUser`**（@Entity / @Table / @Id 等）
- [ ] **3.5 寫 Repository `UserRepository`**（繼承 `JpaRepository<SysUser, Long>`）
- [ ] **3.6 啟動驗證**：應用能起來，沒有「Failed to determine driver」「Cannot connect」之類錯誤

> 此時如果不配 Spring Security，訪問任何 URL 會被默認的 Security 攔成 401，這是正常的。

---

## 階段 4：安全與登錄（Done = 訪問 `/login` 看到登錄頁，能用測試帳號登進去）

- [ ] **4.1 寫 `UserDetailsServiceImpl`**
  - 實現 Spring Security 的 `UserDetailsService` 接口
  - `loadUserByUsername` 中：用 Repository 查 `SysUser`，把它包裝為框架認得的 `UserDetails`
- [ ] **4.2 寫 `SecurityConfig`**
  - 註冊 `BCryptPasswordEncoder` Bean
  - 配置 `SecurityFilterChain`：
    - `/login` 公開
    - 其餘需登錄
    - 啟用 `formLogin`，指定登錄頁 `/login`、成功跳 `/dashboard`、失敗跳 `/login?error`
    - 啟用 `logout`
- [ ] **4.3 寫 `PageController`**：給 `/login` 一個 GET 映射，返回 Thymeleaf 視圖名 `"login"`
- [ ] **4.4 寫 `templates/login.html`**：表單 `action="/login"`、`method="post"`、字段 `username` / `password`
- [ ] **4.5 寫 `DataInitializer`** 啟動時插入 admin/user1/user2，**密碼用 `passwordEncoder.encode()` 加密後再存**
- [ ] **4.6 啟動驗證**
  - 訪問 `/` 自動跳到 `/login`
  - 看到登錄頁
  - 用 `admin / 123456` 能登進去
  - 用錯誤密碼會跳 `/login?error`

> ⚠️ 這是整個項目最容易卡住的階段，看 04-踩坑記錄.md 的 **「Spring Security 類問題」** 段。

---

## 階段 5：頁面與接口（Done = 登錄後能看到 dashboard 和用戶列表）

- [ ] **5.1 在 `PageController` 加 `/dashboard` 的 GET**
  - 讀 `@AuthenticationPrincipal UserDetails`、`HttpSession`、`HttpServletRequest`
  - 把 username、sessionId、loginTime、訪問次數、Cookie 列表塞進 Model
- [ ] **5.2 寫 `templates/dashboard.html`**：用 `th:text`、`th:each` 展示上面的數據
- [ ] **5.3 寫 `UserController`（@RestController）**
  - GET `/api/users` 返回 `List<UserVO>`
  - 用 record 定義 `UserVO(id, username, enabled)`，**故意不返回 password 字段**
- [ ] **5.4 在 `PageController` 加 `/users` 的 GET**，返回 `"users"` 視圖
- [ ] **5.5 寫 `templates/users.html`**：頁面內 `<script>` 用 `fetch('/api/users')` 拿數據渲染表格
- [ ] **5.6 加退出登錄按鈕**：dashboard 上一個 `form action="/logout" method="post"`
- [ ] **5.7 加跳轉鏈接**：dashboard 和 users 互相能跳

---

## 階段 6：完善與驗證（Done = 所有功能正常，理解每個現象背後的原理）

- [ ] **6.1 在 SecurityConfig 加 `rememberMe`**：key、tokenValiditySeconds、rememberMeParameter
- [ ] **6.2 在 login.html 加「記住我」勾選框**：`name="remember-me"`（必須和上面 rememberMeParameter 一致）
- [ ] **6.3 驗證 JSESSIONID Cookie**
  - 登錄後在 dashboard 上能看到 JSESSIONID
  - 瀏覽器開發者工具 Application → Cookies 也能看到
  - 它的「有效期」應顯示「瀏覽器關閉即失效」
- [ ] **6.4 驗證 Remember-me Cookie**
  - 勾選「記住我」登錄，dashboard 應多出一個 `remember-me` Cookie
  - 它的有效期應該是 7 天
  - **關閉瀏覽器再打開**，直接訪問 `/dashboard` 應免登錄
- [ ] **6.5 驗證接口的 Cookie 自動攜帶**
  - 進入 `/users` 頁面，打開 F12 → Network → 找到 `/api/users` 請求
  - 確認 Request Headers 裏有 `Cookie: JSESSIONID=xxx`
  - 確認響應是 JSON，不是 HTML
- [ ] **6.6 驗證 Session 失效後的行為**
  - 退出登錄
  - 直接訪問 `/dashboard` → 應該被攔截跳到 `/login`
  - 在 `/users` 頁面停留，後端重啟後刷新 → fetch 失敗 → 跳回登錄頁

---

## 階段間的依賴關係

```
階段 1 ──▶ 階段 2 ──▶ 階段 3 ──▶ 階段 4 ──▶ 階段 5 ──▶ 階段 6
   │          │          │          │          │
   └──────────┴──────────┴──────────┴──────────┘
              所有階段都依賴前面的階段全部完成
```

**唯一可以並行的：** 階段 5 中的「頁面 / 接口」可以分開做。

## 進度自檢表（每完成一個階段就回來打勾）

| 階段 | 狀態 | 完成標誌（自驗收標準） |
|---|---|---|
| 1. 環境準備 | ☐ | 命令行能查到 java/maven 版本，數據庫客戶端能連庫 |
| 2. 項目骨架 | ☐ | `mvn spring-boot:run` 能啟動，訪問 `localhost:8080` 有響應 |
| 3. 數據層 | ☐ | sys_user 表有 3 條測試數據，啟動沒紅錯 |
| 4. 安全與登錄 | ☐ | admin/123456 能登進去，錯誤密碼會被攔下 |
| 5. 頁面與接口 | ☐ | dashboard 顯示 Session 信息，users 頁面從 API 拉到數據 |
| 6. 完善與驗證 | ☐ | 勾選「記住我」並關閉瀏覽器後，重開仍是登錄狀態 |

全部打勾 → 你已經獨立從 0 搭建了一個完整的 Spring Security 登錄系統。
