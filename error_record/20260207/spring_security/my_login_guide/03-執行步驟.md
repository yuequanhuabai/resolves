## 階段 1 — 環境準備

### 1.1 確認 JDK 17+

打開命令行（CMD 或 Bash），輸入 `java -version`。

- 期望看到 `17` 或更高版本（如 `17.0.x` / `21.0.x`）
- 如果是 8 / 11，需要安裝 JDK 17（推薦 [Eclipse Temurin](https://adoptium.net/) 或 Oracle JDK）
- 安裝後配置 `JAVA_HOME` 環境變量，並把 `%JAVA_HOME%\bin` 加進 `PATH`

### 1.2 確認 Maven

命令行輸入 `mvn -v`。
- 期望看到 Apache Maven 3.6+ 的版本
- 沒有？要麼裝 Maven，要麼用 IDE 自帶的 Maven Wrapper（`mvnw`）

### 1.3 準備 SQL Server

本指南以 SQL Server 為例。如果你用 MySQL/PostgreSQL，原理一樣，只是驅動和 URL 不同。

- 你需要一個能訪問的 SQL Server 實例（本機 / 遠程都可以）
- 拿到 4 個信息：**host / port / 用戶名 / 密碼**
- 用 [DBeaver](https://dbeaver.io/) 或 SSMS 連通它，這一步必須成功

### 1.4 數據庫客戶端能連上

打開 DBeaver → 新建連接 → SQL Server → 填入上面 4 個信息 → Test Connection

**通了再進下一步。** 不要跳過這個確認。

## 階段 2 — 項目骨架

### 2.1 用 Spring Initializr 生成骨架

訪問 [https://start.spring.io](https://start.spring.io)，按以下選擇：

| 配置項 | 推薦值 |
|---|---|
| Project | Maven |
| Language | Java |
| Spring Boot | 3.2.5 |
| Group | com.example |
| Artifact | test-login-back |
| Packaging | Jar |
| Java | 17 |

**Dependencies（點 ADD DEPENDENCIES 添加）：**
- Spring Web
- Spring Security
- Thymeleaf
- Spring Data JPA
- Lombok
- MS SQL Server Driver（如果用其他庫，選對應驅動）

點 GENERATE → 下載 zip → 解壓到目標目錄。

### 2.2 導入 IDE

- IntelliJ IDEA：File → Open → 選 pom.xml → Open as Project
- 等 Maven 把依賴下完（右下角會有進度條，第一次可能要幾分鐘）

### 2.3 啟動類

Initializr 已經幫你生成了 `XxxApplication.java`，裏面有 `@SpringBootApplication` 和 `main` 方法。**先不要動它**。

### 2.4 配置 application.yml

`src/main/resources/` 下默認有 `application.properties`。**把它刪掉，新建 `application.yml`**（YAML 更清晰，是 Spring Boot 推薦的格式）。

只配 `server.port: 8080`，先不配數據庫。

**驗證點：** 此時直接啟動會報 `Failed to configure a DataSource`，因為你選了 JPA 但沒配數據源。**這是正常的**，下一階段就解決。

## 階段 3 — 數據層

### 3.1 建庫建表

在 DBeaver 裏執行 SQL（按你項目的 init.sql 邏輯，但你要自己寫一遍）：

需要做的事：
1. `CREATE DATABASE test_login`
2. `USE test_login`
3. `CREATE TABLE sys_user`
   - `id`：BIGINT，主鍵，自增（SQL Server 用 `IDENTITY(1,1)`）
   - `username`：字符串 50，非空，唯一
   - `password`：字符串 255（要存 BCrypt 哈希，60 字符以上，給 255 保險）
   - `enabled`：BIT，非空，默認 1

**驗證點：** 在 DBeaver 裏執行 `SELECT * FROM sys_user` 不報錯（即使空表）。

### 3.2 配置 application.yml 數據源

需要查的關鍵字：
- `spring.datasource.url`（SQL Server JDBC URL 格式）
- `spring.datasource.driver-class-name`（`com.microsoft.sqlserver.jdbc.SQLServerDriver`）
- `username` / `password`

⚠️ **連接字符串注意點：**
- SQL Server 2017+ 默認啟用加密，URL 中通常需要帶 `encrypt=true;trustServerCertificate=true`
- 庫名用 `databaseName=test_login` 而不是 URL 末尾加路徑

### 3.3 配置 JPA

在 `application.yml` 中加 `spring.jpa.*`：

| 配置 | 值 | 為什麼 |
|---|---|---|
| `hibernate.ddl-auto` | `none` | 表你已經自己建好了，不要讓 Hibernate 來改 |
| `show-sql` | `true` | 控制檯打印 SQL，方便排查 |
| `database-platform` | `org.hibernate.dialect.SQLServerDialect` | 告訴 Hibernate 用 SQL Server 方言 |

### 3.4 寫 SysUser 實體

新建包 `entity`，裏面新建 `SysUser.java`。

需要用到的註解（依序）：
- `@Entity`（告訴 JPA 這是個實體）
- `@Table(name = "sys_user")`（顯式指定表名，不依賴默認規則）
- `@Data`（Lombok，省 getter/setter）
- 字段上：
  - `id` 用 `@Id` + `@GeneratedValue(strategy = GenerationType.IDENTITY)`
  - `username` 用 `@Column(nullable = false, unique = true, length = 50)`
  - `password` 用 `@Column(nullable = false, length = 255)`
  - `enabled` 用 `@Column(nullable = false)`，類型用 `Boolean`，**默認值寫成 `= true`**

**驗證點：** IDE 不報紅，引入正確的 import（`jakarta.persistence.*` 而不是 `javax.persistence.*`，Spring Boot 3 用 Jakarta）。

### 3.5 寫 UserRepository

新建包 `repository`，裏面新建 `UserRepository.java`。

關鍵點：
- 它是個 **interface**，不是 class
- 繼承 `JpaRepository<SysUser, Long>`（兩個泛型分別是 實體類型 / 主鍵類型）
- 加一個方法 `Optional<SysUser> findByUsername(String username)`
- **不需要寫實現！** Spring Data JPA 會根據方法名自動生成 SQL `WHERE username = ?`

### 3.6 啟動驗證

跑 `mvn spring-boot:run`（或 IDE 的綠色三角）。

**期望現象：**
- 控制檯有 `Tomcat started on port 8080` 之類的字樣
- 沒有紅色報錯
- 訪問 `http://localhost:8080/` 會跳到一個默認的 Spring Security 登錄頁（這是 Spring Security 自動配置的，下一階段會替換掉）

**如果報錯：** 看 04-踩坑記錄.md 的「數據源連不上」段。

## 階段 4 — 安全與登錄

### 4.1 寫 UserDetailsServiceImpl

新建包 `service`，裏面新建 `UserDetailsServiceImpl.java`。

要做的事：
- 加 `@Service` 註解
- 實現 `org.springframework.security.core.userdetails.UserDetailsService` 接口
- 構造器注入 `UserRepository`
- 實現 `loadUserByUsername(String username)`：
  - 用 `userRepository.findByUsername(username)` 查
  - 找不到？拋 `UsernameNotFoundException`
  - 找到了？用 `org.springframework.security.core.userdetails.User.builder()` 構造一個 `UserDetails`：
    - `.username(...)` `.password(...)`（直接傳數據庫裏的密文，**不要在這裏解密**）
    - `.disabled(!sysUser.getEnabled())`（注意取反）
    - `.roles("USER")`

**為什麼這麼設計：** Spring Security 不認識你的 `SysUser`，但它認識 `UserDetails` 接口。這個 Service 就是「翻譯官」。

### 4.2 寫 SecurityConfig

新建包 `config`，裏面新建 `SecurityConfig.java`。

註解：
- `@Configuration`
- `@EnableWebSecurity`

需要寫兩個 `@Bean`：

**Bean 1：`PasswordEncoder`**
- 返回 `new BCryptPasswordEncoder()`
- 為什麼要這個？因為 Spring Security 校驗密碼時會用它的 `matches(明文, 密文)` 方法

**Bean 2：`SecurityFilterChain filterChain(HttpSecurity http)`**
- 這是 Spring Security 6+ 的新寫法（不再繼承 `WebSecurityConfigurerAdapter`）
- 配置流式 API：
  - `.authorizeHttpRequests(auth -> auth.requestMatchers("/login").permitAll().anyRequest().authenticated())`
  - `.formLogin(form -> form.loginPage("/login").defaultSuccessUrl("/dashboard", true).failureUrl("/login?error").permitAll())`
  - `.logout(logout -> logout.logoutUrl("/logout").logoutSuccessUrl("/login?logout").permitAll())`
- 最後 `return http.build();`

**坑點預告：** `defaultSuccessUrl` 第二個參數 `true` 表示「總是跳這裏」。如果不傳 `true`，登錄前訪問過受保護頁面的話會跳到那裏。為了 Demo 體驗一致，傳 `true`。

### 4.3 PageController 處理 /login

新建包 `controller`，裏面新建 `PageController.java`。

- 加 `@Controller`（**不是 `@RestController`**，因為要返回 HTML 視圖）
- 加方法 `@GetMapping("/login") public String loginPage()`，**直接 `return "login";`**
- 這個字符串 `"login"` 會被 Thymeleaf 解析為 `templates/login.html`

### 4.4 templates/login.html

`src/main/resources/templates/` 下新建 `login.html`。

關鍵點：
- HTML 標籤加 `xmlns:th="http://www.thymeleaf.org"`（聲明 Thymeleaf 命名空間）
- 寫一個 `<form th:action="@{/login}" method="post">`
- 兩個 input：`name="username"` 和 `name="password"`
  - **字段名必須是這兩個**（除非你在 SecurityConfig 顯式改）
- 一個 submit button
- （可選）用 `th:if="${param.error}"` 顯示錯誤提示

### 4.5 DataInitializer 造測試數據

新建包 `init`，新建 `DataInitializer.java`。

關鍵點：
- 加 `@Component`
- 實現 `org.springframework.boot.CommandLineRunner` 接口
- 構造器注入 `UserRepository` 和 `PasswordEncoder`
- `run` 方法裏：
  - 先用 `findByUsername` 判斷用戶是否已存在（避免重複插）
  - 不存在就 new 一個 `SysUser`，**密碼必須用 `passwordEncoder.encode("123456")` 加密後再 set**
  - `userRepository.save(u)`

⚠️ **最常見的坑：** 不加密直接存明文。後果是登錄永遠失敗，因為 Spring Security 期望密文。

### 4.6 啟動驗證

重啟應用。

**期望現象：**
- 控制檯有「[DataInitializer] 創建用戶: admin」等日誌
- 訪問 `http://localhost:8080/dashboard` 自動 302 跳到 `/login`
- 登錄頁是你寫的那個（不是 Spring Security 默認的）
- 用 `admin / 123456` 登錄成功跳到 `/dashboard`（此時 dashboard 還沒實現，會 404 或白頁，正常）
- 用 `admin / 錯誤密碼` 跳回 `/login?error`，顯示錯誤提示

## 階段 5 — 頁面與接口

### 5.1 PageController 加 /dashboard

在 `PageController` 裏加新方法：

`@GetMapping("/dashboard")` 方法簽名建議：
```
public String dashboard(@AuthenticationPrincipal UserDetails userDetails,
                        HttpSession session,
                        HttpServletRequest request,
                        Model model)
```

業務邏輯（自己實現，提示如下）：
1. 用 `session.getAttribute("loginTime")` 判空，第一次設置為當前時間（用 `LocalDateTime` + `DateTimeFormatter`）
2. 訪問次數計數器：每次刷新從 session 取出 `visitCount`，+1 後存回去
3. 把 username、sessionId、loginTime、visitCount、`session.getMaxInactiveInterval()` 都 `model.addAttribute`
4. 從 `request.getCookies()` 拿到 Cookie 數組，遍歷後封裝成 `List<Map<String,String>>`，也塞進 model
5. `return "dashboard"`

### 5.2 templates/dashboard.html

關鍵 Thymeleaf 用法：
- `th:text="${username}"` — 渲染變量
- `th:each="c : ${cookies}"` — 遍歷集合
- `th:if="${...}"` — 條件渲染

頁面結構建議：
- 頂部一個 header（顯示標題、退出按鈕、跳轉用戶列表的鏈接）
- 一個卡片顯示 Session 信息（用戶名、Session ID、登錄時間、訪問次數、超時時間）
- 一個卡片顯示 Cookie 表格（name / value / maxAge / path / httpOnly / secure）
- 退出按鈕用一個 `<form th:action="@{/logout}" method="post">` 包一個 button

### 5.3 UserController（REST 接口）

在 `controller` 包下新建 `UserController.java`。

- 加 `@RestController`（**這個會自動把返回值序列化為 JSON**）
- 加 `@RequestMapping("/api")`
- 構造器注入 `UserRepository`
- 方法 `@GetMapping("/users")`：
  - `userRepository.findAll().stream().map(...).toList()`
  - map 的目標是 `UserVO`，**不要直接返回 `SysUser`**（會把 password 字段也序列化出去！）

新建 `UserVO`，可以用 Java record 一行搞定：
```
public record UserVO(Long id, String username, Boolean enabled) {}
```

⚠️ **這是這個項目隱含的安全教訓**：永遠不要把實體類直接序列化返給前端。用 DTO/VO 顯式控制字段。

### 5.4 PageController 加 /users 視圖

在 `PageController` 加 `@GetMapping("/users")` 返回 `"users"`。**不要把 `/api/users` 和 `/users` 混為一談**：
- `/users` → 返回 HTML 頁面（殼子）
- `/api/users` → 返回 JSON 數據

### 5.5 templates/users.html

頁面結構：
- 一個空表格（thead 寫好，tbody 留空）
- 頁面底部一段 `<script>`，用 `fetch('/api/users')` 拿數據，動態渲染表格
- fetch 的 `.then(res => ...)` 中要判斷 `res.ok`：未登錄時 Spring Security 會 302 跳轉，瀏覽器跟隨後得到登錄頁 HTML（res.ok 會是 false 或 Content-Type 不對）
  - 簡化處理：`if (!res.ok) window.location.href = '/login'`

### 5.6 退出登錄

在 dashboard.html 加：
```html
<form th:action="@{/logout}" method="post">
  <button type="submit">退出登錄</button>
</form>
```

⚠️ **必須是 POST**，不能是 GET。Spring Security 默認只接受 POST 退出（CSRF 保護）。

### 5.7 互相跳轉鏈接

dashboard.html 上加 `<a href="/users">用戶列表</a>`，
users.html 上加 `<a href="/dashboard">返回首頁</a>`。

## 階段 6 — 完善與驗證

### 6.1 開啟 Remember-me

在 `SecurityConfig` 的 `filterChain` 中加 `.rememberMe(remember -> ...)`，配置三項：

- `.key("一段足夠隨機的字符串")` — 簽名密鑰，**生產環境必須保密且每環境不同**
- `.tokenValiditySeconds(7 * 24 * 60 * 60)` — 7 天，單位秒
- `.rememberMeParameter("remember-me")` — 表單字段名

### 6.2 登錄頁加勾選框

在 login.html 的表單裏加：
```html
<input type="checkbox" name="remember-me"/>
<label>記住我（7天內免登錄）</label>
```

⚠️ `name` **必須**和上面 `rememberMeParameter` 配置的一致。

### 6.3-6.6 自驗收

按 02 文檔階段 6 的 checklist 一條條做。重點是：

- 打開 F12 看 Network 面板，確認請求頭有 Cookie
- 打開 Application → Cookies 看兩個 Cookie：JSESSIONID（Session）、remember-me（持久化）
- 關閉瀏覽器再開，是測試 Remember-me 的關鍵
