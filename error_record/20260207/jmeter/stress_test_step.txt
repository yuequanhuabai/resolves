好了，我看到有 SSO 登錄接口（GET /system/auth/loginSso?userName=xxx）不需要密碼。下面是完整的 JMeter 配置步驟。
  ---                                                                                                                                                                                                 JMeter 批量發起流程 - 完整步驟                                                                                                                                                                                                                                                                                                                                                                          前提

  CSV 文件 buylist_init.csv 已導出，格式如下：
  business_id,record_version
  LIST_STRESS_1,0
  LIST_STRESS_2,0
  ...

  JMeter 結構總覽

  Test Plan
  ├── User Defined Variables          ← 全局變量
  ├── Thread Group 1: 登錄獲取Token    ← 1個線程，只跑1次
  │   ├── HTTP Request: 登錄
  │   └── JSON Extractor: 提取 token
  │   └── BeanShell PostProcessor: 設為全局變量
  ├── Thread Group 2: 批量發起流程     ← N個線程併發
  │   ├── CSV Data Set Config
  │   ├── HTTP Header Manager
  │   └── HTTP Request: updateNew
  └── View Results Tree / Summary Report

  ---
  第一步：新建 Test Plan

  打開 JMeter → 新建 Test Plan

  添加全局變量 (Test Plan → 右鍵 → Add → Config Element → User Defined Variables)：

  ┌────────────┬──────────────┐
  │    Name    │    Value     │
  ├────────────┼──────────────┤
  │ BASE_URL   │ 你的服務器IP │
  ├────────────┼──────────────┤
  │ PORT       │ 9600         │
  ├────────────┼──────────────┤
  │ MAKER_USER │ maker帳號    │
  ├────────────┼──────────────┤
  │ MAKER_PWD  │ maker密碼    │
  └────────────┴──────────────┘

  ---
  第二步：Thread Group 1 - 登錄獲取 Token

  右鍵 Test Plan → Add → Threads → Thread Group

  - Name: 登錄獲取Token
  - Number of Threads: 1
  - Loop Count: 1

  2.1 HTTP Request - 登錄

  右鍵 Thread Group → Add → Sampler → HTTP Request

  ┌──────────────┬───────────────────────────────┐
  │    配置項    │              值               │
  ├──────────────┼───────────────────────────────┤
  │ Protocol     │ http                          │
  ├──────────────┼───────────────────────────────┤
  │ Server Name  │ ${BASE_URL}                   │
  ├──────────────┼───────────────────────────────┤
  │ Port         │ ${PORT}                       │
  ├──────────────┼───────────────────────────────┤
  │ Method       │ POST                          │
  ├──────────────┼───────────────────────────────┤
  │ Path         │ /admin-api/system/auth/login  │
  ├──────────────┼───────────────────────────────┤
  │ Body Data    │ 見下方                        │
  ├──────────────┼───────────────────────────────┤
  │ Content-Type │ 在 HTTP Header Manager 中設置 │
  └──────────────┴───────────────────────────────┘

  先加 HTTP Header Manager（右鍵 HTTP Request → Add → Config Element → HTTP Header Manager）：

  ┌──────────────┬──────────────────┐
  │     Name     │      Value       │
  ├──────────────┼──────────────────┤
  │ Content-Type │ application/json │
  ├──────────────┼──────────────────┤
  │ tenant-id    │ 1                │
  └──────────────┴──────────────────┘

  Body Data：
  {
    "username": "${MAKER_USER}",
    "password": "${MAKER_PWD}"
  }

  如果你的環境支持 SSO 免密登錄，也可以改用 GET /admin-api/system/auth/loginSso?userName=${MAKER_USER}

  2.2 JSON Extractor - 提取 Token

  右鍵 HTTP Request → Add → Post Processors → JSON Extractor

  ┌────────────────────────────┬────────────────────┐
  │           配置項           │         值         │
  ├────────────────────────────┼────────────────────┤
  │ Names of created variables │ access_token       │
  ├────────────────────────────┼────────────────────┤
  │ JSON Path expressions      │ $.data.accessToken │
  ├────────────────────────────┼────────────────────┤
  │ Default Values             │ TOKEN_NOT_FOUND    │
  └────────────────────────────┴────────────────────┘

  2.3 BeanShell PostProcessor - 設為全局變量

  右鍵 HTTP Request → Add → Post Processors → BeanShell PostProcessor

  String token = vars.get("access_token");
  props.put("ACCESS_TOKEN", token);
  log.info("=== Token obtained: " + token);

  ---
  第三步：Thread Group 2 - 批量發起流程

  右鍵 Test Plan → Add → Threads → Thread Group

  - Name: 批量發起流程
  - Number of Threads: 50（併發數，按需調整）
  - Ramp-Up Period: 10（10秒內啟動所有線程）
  - Loop Count: 1
  - Startup delay: 3（等登錄完成）

  3.1 CSV Data Set Config

  右鍵 Thread Group → Add → Config Element → CSV Data Set Config

  ┌────────────────────┬───────────────────────────────────────────┐
  │       配置項       │                    值                     │
  ├────────────────────┼───────────────────────────────────────────┤
  │ Filename           │ C:/jmeter/buylist_init.csv（你的CSV路徑） │
  ├────────────────────┼───────────────────────────────────────────┤
  │ File Encoding      │ UTF-8                                     │
  ├────────────────────┼───────────────────────────────────────────┤
  │ Variable Names     │ business_id,record_version                │
  ├────────────────────┼───────────────────────────────────────────┤
  │ Delimiter          │ ,                                         │
  ├────────────────────┼───────────────────────────────────────────┤
  │ Ignore first line  │ True（如果有表頭）                        │
  ├────────────────────┼───────────────────────────────────────────┤
  │ Recycle on EOF     │ False                                     │
  ├────────────────────┼───────────────────────────────────────────┤
  │ Stop thread on EOF │ True                                      │
  ├────────────────────┼───────────────────────────────────────────┤
  │ Sharing mode       │ All threads                               │
  └────────────────────┴───────────────────────────────────────────┘

  3.2 HTTP Header Manager

  右鍵 Thread Group → Add → Config Element → HTTP Header Manager

  ┌───────────────┬────────────────────────────────────┐
  │     Name      │               Value                │
  ├───────────────┼────────────────────────────────────┤
  │ Content-Type  │ application/json                   │
  ├───────────────┼────────────────────────────────────┤
  │ Authorization │ Bearer ${__property(ACCESS_TOKEN)} │
  ├───────────────┼────────────────────────────────────┤
  │ tenant-id     │ 1                                  │
  └───────────────┴────────────────────────────────────┘

  3.3 HTTP Request - 發起流程

  右鍵 Thread Group → Add → Sampler → HTTP Request

  ┌─────────────┬──────────────────────────────┐
  │   配置項    │              值              │
  ├─────────────┼──────────────────────────────┤
  │ Protocol    │ http                         │
  ├─────────────┼──────────────────────────────┤
  │ Server Name │ ${BASE_URL}                  │
  ├─────────────┼──────────────────────────────┤
  │ Port        │ ${PORT}                      │
  ├─────────────┼──────────────────────────────┤
  │ Method      │ PUT                          │
  ├─────────────┼──────────────────────────────┤
  │ Path        │ /admin-api/buyList/updateNew │
  └─────────────┴──────────────────────────────┘

  Body Data：
  [{
    "id": null,
    "buyListId": null,
    "businessId": "${business_id}",
    "assetType": "Bond",
    "productCode": "XS937663002",
    "clientId": "",
    "approvalStatus": 0,
    "clientCount": 1,
    "hasMultipleClients": false,
    "productCodeError": false,
    "recordVersion": ${record_version},
    "businessType": 1
  }]

  productCode 用你環境中 SECURITY_MASTER 裡已有的值。

  3.4 Response Assertion（可選）

  右鍵 HTTP Request → Add → Assertions → Response Assertion

  - Field to Test: Response Code
  - Pattern Matching Rules: Equals
  - Patterns to Test: 200

  3.5 JSON Assertion（可選）

  右鍵 HTTP Request → Add → Assertions → JSON Assertion

  - JSON Path: $.data.valid
  - Expected Value: true

  ---
  第四步：添加監聽器

  右鍵 Test Plan → Add → Listener：

  - View Results Tree — 調試時用，看每個請求的詳細結果
  - Summary Report — 看整體吞吐量、平均響應時間、錯誤率
  - Aggregate Report — 更詳細的統計

  ---
  第五步：執行順序控制

  確保 Thread Group 1 先於 Thread Group 2 執行：

  Test Plan 勾選 "Run Thread Groups consecutively (i.e. one at a time)"

  ---
  運行後驗證

  執行完畢後，確認流程都已創建成功：

  -- 檢查有多少流程實例被創建
  SELECT COUNT(*) FROM IHUB_BUY_LIST
  WHERE business_id LIKE 'LIST_STRESS_%'
    AND approval_status = 1
    AND biz_status IN (1, 4);

  -- 檢查 ACT_RU_TASK 中待審批任務數量
  SELECT COUNT(*) FROM ACT_RU_TASK
  WHERE ASSIGNEE_ = 'approve001的用戶ID';

  確認任務數量正確後，再導出 task ID 進入審批壓測階段。
