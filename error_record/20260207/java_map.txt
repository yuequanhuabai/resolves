● Java的知识地图，我按层次来呈现，从底层到上层：

  ---
  第一层：语言本身

  ▎ 这是一切的地基

  - 基础语法：数据类型、控制流、运算符
  - 面向对象：封装、继承、多态、抽象类、接口
  - 高级语言特性：泛型、注解、异常处理、Lambda、Stream

  ---
  第二层：JVM（Java虚拟机）

  ▎ Java程序运行的底层引擎，理解它才能理解"为什么"

  - 内存模型：堆、栈、方法区的划分
  - 垃圾回收：GC机制、GC算法、GC调优
  - 类加载机制：双亲委派模型
  - JVM调优：内存参数、性能分析

  ---
  第三层：核心标准库

  ▎ Java自带的工具箱

  - 集合框架：List、Map、Set及其实现原理
  - 并发编程：线程、锁、线程池、并发容器、CompletableFuture
  - IO/NIO：文件读写、网络通信
  - 反射：运行时动态操作类

  ---
  第四层：Spring生态

  ▎ Java企业开发的核心框架，现代Java开发几乎离不开它

  - Spring Core：IoC容器、依赖注入、AOP
  - Spring Boot：快速构建应用、自动配置
  - Spring MVC：Web开发、RESTful API
  - Spring Data：数据库访问简化
  - Spring Security：认证与授权

  ---
  第五层：数据存储

  ▎ 数据从哪来、往哪存

  - 关系型数据库：JDBC、MyBatis、JPA/Hibernate、事务
  - 缓存：Redis、本地缓存（Caffeine）
  - 连接池：HikariCP、Druid

  ---
  第六层：分布式与微服务

  ▎ 系统变大之后要解决的问题

  - Spring Cloud生态：服务注册发现、负载均衡、熔断、网关
  - 消息队列：Kafka、RabbitMQ（异步解耦）
  - 分布式理论：CAP、BASE、分布式事务

  ---
  贯穿所有层的横切面

  ┌──────────────┬──────────────────────────────────┐
  │    横切面    │               内容               │
  ├──────────────┼──────────────────────────────────┤
  │ 设计模式     │ 单例、工厂、代理、观察者等23种   │
  ├──────────────┼──────────────────────────────────┤
  │ 工程化       │ Maven/Gradle、Git、Docker、CI/CD │
  ├──────────────┼──────────────────────────────────┤
  │ 性能与稳定性 │ 限流、熔断、监控、日志           │
  ├──────────────┼──────────────────────────────────┤
  │ 网络基础     │ HTTP、TCP/IP、Netty              │
  └──────────────┴──────────────────────────────────┘

  ---
  整体结构图

  微服务/分布式
        ↑
  Spring生态 + 数据存储
        ↑
  核心标准库（集合/并发/IO）
        ↑
  JVM（运行引擎）
        ↑
  Java语言基础
